Single Package Mode

只生成一个包，先交互补齐，再输出 Word

先上传招标文件，系统只针对你选中的包生成待填写项。重复字段只问一次， 页面只保留在线交互填写，再下载最终文档。

流程

  • 上传招标文件并识别包件
  • 选择一个目标包生成待填写项
  • 页面交互填写待补字段
  • 只生成当前包的 Word 文档

0. API 密钥

直接在页面填写；浏览器会自动记住上一次输入，仅保存在当前设备的 localStorage。

此页面直接透传你填写的密钥；不再依赖服务端 `.env` 中的 LLM_API_KEY。

1. 上传并解析

上传后先做解析，不会立即生成全部包。

2. 选择目标包

后续只会生成你当前选中的一个包。

3. 生成偏好

可自定义目录顺序、章节编号风格、字体与语言风格。目录顺序建议只写章节名称，不写编号；未列出的章节会自动排在后面。

章节数量、顺序和嵌套结构请在下面的“本次将生成的章节结构”区域里直接拖动调整。

4. 生成文档

系统会把你填写的答案回填到当前包的所有对应位置。
