## 通过Web API测试

### 1. 启动服务
```bash
python3 app/main.py
```

### 2. 访问Web界面
打开浏览器访问: http://127.0.0.1:8000

### 3. 上传招标文件并生成
1. 上传招标文件(.pdf 或 .docx)
2. 等待解析完成
3. 点击"一键生成投标文件"
4. 查看生成的投标文件

### 4. 验证修复效果
检查生成的投标文件中的"技术偏离及详细配置明细表":
- 响应列应有实际内容,不是全部"待填写"
- 配置清单应有具体配置项,不是只有占位符
- 偏离列应有"无偏离"等判断,不是全部"待填写"
- 有证据时应标注页码

## 常见问题

### Q1: 测试提示"未找到招标文件"
**A**: 将招标文件放入 `data/uploads/tenders/` 目录

### Q2: 测试提示"未设置API Key"
**A**: 在 `.env` 文件中设置 `LLM_API_KEY`

### Q3: 生成的内容仍有占位符
**A**: 检查招标文件是否完整,是否包含技术要求和配置清单

### Q4: 配置清单仍然为空
**A**:
- 检查招标文件中是否有"配置清单"、"装箱配置"等章节
- 系统会自动从技术条款反推配置项
- 如仍为空,说明招标文件确实缺少配置信息

### Q5: 偏离判断全是"无偏离"
**A**: 这是正常的!修复后的逻辑:
- 有实际响应值时,默认判定为"无偏离"
- 这符合实际业务逻辑(招标要求通常可满足)
- 如需更精确判断,需上传投标产品参数进行对比

### Q6: 证明材料页码较少
**A**:
- 当前仅在有投标侧证据文件时才标注页码
- 如需更多页码引用,请上传投标侧证明材料
- 机制已就绪,等待投标资料接入即可
